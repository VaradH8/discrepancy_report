Registered changes, tagged global vs local.

Keeps only equipment in the master register (Equipment List to Inventive) from your comparison. If you've also run the cross-reference, each change is rewritten as GLOBALLY / LOCALLY ADDED or GLOBALLY / LOCALLY REMOVED — global = found nowhere else, local = exists on other sheets. The cross-reference is optional; without it, rows keep plain ADDED / REMOVED.

Whole folders in, registered reports out.

Pick the issued-revision folder and the new-revision folder. Sheets are paired by number (SHT-01 ↔ SHT-01 …); each pair is compared against the master equipment register: GLOBALLY ADDED = registered equipment new in the extracted set; LOCALLY ADDED = unregistered tag found on another extracted sheet too; REMOVED 1 = unregistered tag gone from the extracted set; REMOVED = registered equipment gone from the extracted set (the critical case). Register entries missing from both the issued and extracted sets across the whole batch are reported once as Removed 2. You get one zip with a Registered + unfiltered Changes excel per sheet.
